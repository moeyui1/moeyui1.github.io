曾经本博客用的也是 `.tk` 的免费域名，毕竟是个免费域名。直到后来买了腾讯云的一块钱 `.cn` 域名之后，才换了一个在一般人眼中看起来稍微正常点的域名。原来的 `.tk` 还曾经被老外吐槽过。。。

![老外图](https://oeoiy7i1f.qnssl.com/%E6%95%99%E8%82%B2%E4%BC%98%E6%83%A0/%E8%80%81%E5%A4%96%E5%90%90%E6%A7%BD.tk.PNG)

当时一直不明白为什么`.tk`会被认为是含 malicious 内容的不安全网站。直到最近接了一个网站项目……

<!--more-->

客户要求做一个网站，没有域名要求，本来是打算直接 ip 访问的，但是手上正好那么多免费的`.tk`域名，干脆随便用一个吧。

上线之后，一些用户（约15%）反映无法访问网站`(っﾟДﾟ)っ!! `。因为网站是用 react 写的，吾辈首先想到的是浏览器的兼容性问题。

然而，，，结果怎样呢？吾辈放几张图大家感受下：

这是`.cn`的 ping 情况：

![.cn的ping](https://oeoiy7i1f.qnssl.com/%E6%BB%9A%E5%90%A7%20.tk/cn%E7%9A%84ping%E7%BB%93%E6%9E%9C.png)

这是`.tk`的 ping 情况：

![tk的ping情况1](https://oeoiy7i1f.qnssl.com/%E6%BB%9A%E5%90%A7%20.tk/tk%E7%9A%84ping%E7%BB%93%E6%9E%9C1.png)

没有延迟数据的地区情况大致如下：

![tk的ping情况2](https://oeoiy7i1f.qnssl.com/%E6%BB%9A%E5%90%A7%20.tk/tk%E7%9A%84ping%E7%BB%93%E6%9E%9C2.png)

总结一下，大多数电信线路无法解析`.tk`域名，问题明显与DNS有关。同样是托管在 dnspod 上，`.cn`却安然无恙。也就是说，`.tk`可能被电信认为是有害的网站，所以没有解析，这也许就是免费域名的弊端吧。

果然天下是没有免费的午餐的，不过电信这 dns 做得有点绝啊。这次的 bug 通过更换域名算是解决了，姑且为还在用免费域名的站长提个醒吧。网站追求的更多是稳定，一个`.cn`域名一年也就几十块钱，还不到服务器一个月的租金，有条件的还是不要用免费啦。
